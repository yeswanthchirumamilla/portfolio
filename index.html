Spam Classification with ANN

×

Project Overview

An end-to-end spam classification system using Natural Language Processing and Artificial Neural Networks to accurately identify spam messages in SMS and email communications.

Model Performance

  • Accuracy: Achieved 97.2% accuracy on test dataset
  • Precision: 96.8% precision for spam detection
  • Recall: 95.4% recall ensuring minimal false negatives
  • F1-Score: 96.1% balanced performance metric

Data Preprocessing Pipeline

  • Text Cleaning: Removal of special characters, URLs, and numbers
  • Tokenization: Breaking text into individual words
  • Stemming: Reducing words to their root forms using Porter Stemmer
  • Stopword Removal: Filtering out common words with low predictive value
  • Vectorization: Converting text to numerical features using Bag-of-Words

Neural Network Architecture

  • Input Layer: 5000 features from TF-IDF vectorization
  • Hidden Layers: 3 dense layers with ReLU activation (128, 64, 32 neurons)
  • Dropout: 0.3 dropout rate to prevent overfitting
  • Output Layer: Single neuron with sigmoid activation for binary classification
  • Optimizer: Adam optimizer with learning rate scheduling

Dataset and Features

  • SMS Spam Collection Dataset with 5,574 messages
  • Balanced dataset with 4,825 ham and 747 spam messages
  • Feature engineering including message length, word count, and character frequency
  • Cross-validation with stratified k-fold for robust evaluation

Technical Implementation

  • Libraries: TensorFlow/Keras for neural networks, NLTK for NLP
  • Data Handling: Pandas for data manipulation, NumPy for numerical operations
  • Visualization: Matplotlib and Seaborn for performance analysis
  • Model Persistence: Saved trained model and vectorizer for deployment

Speech Recognition Systems: Wake Word Detection Comparison

×

Project Overview

This project explores and compares two different approaches to wake word detection in speech recognition systems. It evaluates a lightweight, real-time streaming model (Porcupine) and a more computationally intensive chunk-based speech-to-text model (Vosk). The goal was to measure their performance, efficiency, and suitability for edge device applications.

Key Features

  • Porcupine (by Picovoice):
    • Real-time, low-latency wake word detection
    • Consumes <4% CPU and ~25 MB RAM
    • Ideal for resource-constrained environments like Raspberry Pi or mobile devices
  • Vosk:
    • General-purpose speech-to-text engine
    • Operates on 2-second audio chunks
    • Consumes >20% CPU and ~180 MB RAM
    • More flexible but less efficient for continuous wake word detection

Technologies Used

  • Python
  • Picovoice Porcupine SDK
  • Vosk API
  • NumPy, SoundDevice, PyAudio
  • CPU and memory benchmarking

Outcome

  • Porcupine proved significantly more efficient for wake word detection on edge devices.
  • Vosk showed better general speech transcription capabilities but is not optimized for real-time keyword spotting.
